# Обґрунтування приналежності проекту до галузі "Комп'ютерні науки"

Для успішного захисту проекту на науковій конференції важливо чітко розмежувати інженерну (програмну) складову та науково-обчислювальну. 

В той час як "Інженерія програмного забезпечення" (ІПЗ) фокусується на архітектурі, патернах проектування та життєвому циклі розробки (наприклад, використання фреймворку Django), спеціальність **"Комп'ютерні науки" (КН)** вивчає алгоритми, обробку даних, математичне моделювання та штучний інтелект.

Проект **Invest Pro 2.0** є не просто веб-додатком, а **системою підтримки прийняття рішень (Decision Support System - DSS)**. Його наукова цінність полягає в алгоритмічній складовій, що базується на наступних 4 стовпах:

## 1. Математичне моделювання прийняття рішень (Алгоритм AHP)
Проект використовує метод аналізу ієрархій (Analytic Hierarchy Process - AHP), розроблений Томасом Сааті. 
Програмно реалізовано алгоритм переведення лінгвістичних оцінок користувача (повзунки UI) у кількісну шкалу. Алгоритм динамічно генерує матрицю попарних порівнянь для різних критеріїв (ризик, прибутковість, вартість, дивіденди), знаходить суми стовпців та обчислює нормалізовані власні вектори (ваги). Це класична прикладна математика, переведена в обчислювальний алгоритм для вирішення багатокритеріальної задачі в умовах невизначеності.

## 2. Обробка природної мови та Штучний інтелект (NLP Sentiment Analysis)
Інтеграція модулів штучного інтелекту є ключовою ознакою проектів з комп'ютерних наук. 
У проекті реалізовано алгоритм збору неструктурованих даних (текстів новин) у реальному часі. Ці дані проходять через лексико-евристичну модель (VADER - Valence Aware Dictionary and sEntiment Reasoner), яка адаптована для мікроблогів та коротких фінансових новин. Система трансформує лінгвістичний контекст у кількісні показники (compound score від -1 до 1), автоматизуючи процес оцінки настроїв ринку.

## 3. Нормалізація гетерогенних даних та Ранжування
Алгоритм ранжування (`rank_stocks`) вирішує проблему зведення різних типів даних до єдиної обчислювальної площини (probability space).
Фінансові метрики поділяються на:
- **Стимулятори** (напр., маржа прибутку, дивіденди) — де більше значення є кращим.
- **Дестимулятори** (напр., Beta-ризик, P/E) — де менше значення є кращим.
Алгоритм використовує математичні перетворення (наприклад, обернені пропорції `1 / (value + 0.01)` для уникнення ділення на нуль), щоб нормалізувати ці дані, після чого застосовує векторне множення (dot product) матриці показників на вектор ваг AHP.

## 4. Алгоритмічна оптимізація та обробка часових рядів
Алгоритм обробляє фінансові часові ряди (time-series data) в реальному часі. Для оптимізації обчислювального навантаження (зниження Time Complexity) та управління лімітами зовнішніх API, впроваджено стратегії In-Memory кешування. Це вирішує комп'ютерну проблему ефективного управління ресурсами при потоковій обробці даних.
